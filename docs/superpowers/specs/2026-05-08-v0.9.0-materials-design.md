# UnrealClaudeMCP v0.9.0 — Materials authoring (instances)

**Date:** 2026-05-08
**Status:** Approved (autonomous-mode design — user-authorized "do everything" execution; spec stands as the design contract)
**Author:** Claude (with Najem)
**Target release:** v0.9.0
**Builds on:** v0.8.0 (28 handlers; sequencer scaffolding)
**Scope:** Four new MCP handlers for `UMaterialInstanceConstant` authoring and material introspection. Full material-graph editing (creating `UMaterialExpression` nodes, wiring them, recompiling shaders) is explicitly out of scope and remains accessible via `execute_unreal_python`.

---

## Goal

Give the LLM **parametric material customization** — the same workflow a tech artist uses 95% of the time: take a parent material, create an instance, override scalar/vector/texture parameters, save. Plus the introspection needed to know what parameters are available before authoring.

Tool count: 28 → 32. Version: 0.8.0 → 0.9.0. **This is the final bundle in the originally-committed roadmap** (v0.4.0 through v0.9.0).

The deliberate scope choice: material graphs are a different beast. Each `UMaterialExpression` subclass (~200+ in UE 5.7) has its own input/output pins, type rules, and editor metadata. Wrapping that in MCP handlers is a multi-month project. Material **instances**, by contrast, expose a clean parameter-override surface that maps cleanly to JSON. This is where the LLM's leverage is highest for the smallest plugin surface.

---

## Scope

### In scope

- `create_material_instance` handler — create a `UMaterialInstanceConstant` asset and set its parent.
- `set_mi_parameter` handler — override a scalar/vector/texture parameter on a material instance.
- `inspect_material` handler — list scalar/vector/texture/static-switch parameter **names** declared by any `UMaterialInterface` (works on both `UMaterial` and `UMaterialInstance`).
- `inspect_material_instance` handler — read the parent material + currently-overridden parameter values from a `UMaterialInstanceConstant`.
- Build.cs addition: `MaterialEditor` (private). Single new module dependency, lightweight compared to v0.8.0's 4-module change.
- Version bumps (.uplugin, mcp_manifest.json, bridge, README).
- Bridge catalog + JSON Schema for the four new handlers.
- Bridge unit tests: schema-shape tests + manifest sync count.
- Smoke step: read-only — find any `Material` in the project, call `inspect_material`, verify the result shape.
- `docs/TOOLS.md` additions.
- Spec doc (this file).

### Out of scope

- Material-graph authoring (`UMaterialExpression` add/wire/compile). Defer to v0.9.x or use `execute_unreal_python`. The graph API is large enough to warrant its own bundle if ever pursued.
- `UMaterialInstanceDynamic` (runtime-only MIs created at gameplay time). v0.9.0 only handles editor-time `UMaterialInstanceConstant`.
- Material function instances and material layer parameters (`EMaterialParameterAssociation::LayerParameter` / `BlendParameter`). v0.9.0 only handles `GlobalParameter` (the default association). Layered materials are an advanced workflow and add a parameter-association axis to every call.
- Static-switch parameters (`SetMaterialInstanceStaticSwitchParameterValue`). These trigger shader recompiles and have wider side-effect surface. Inspection lists them via `inspect_material`'s static-switch list, but mutation is deferred.
- `FDoubleVectorParameterValue`, `FRuntimeVirtualTextureParameterValue`, `FSparseVolumeTextureParameterValue` — niche parameter types; defer.
- Material **deletion** is already covered by v0.7.0's `delete_asset` (with referencer safety check).

## Handler: `create_material_instance`

### Parameters

| Param | Type | Required | Default |
|---|---|---|---|
| `parent_path` | string | Yes | — |
| `path` | string | Yes | — |
| `name` | string | Yes | — |

`parent_path` is the source `UMaterial` or `UMaterialInstance` asset path. `path` is the destination folder under `/Game/`. `name` is the leaf asset name.

### Result

```json
{
  "ok": true,
  "asset_path": "/Game/Materials/MI_Stone.MI_Stone",
  "package_path": "/Game/Materials/MI_Stone",
  "parent_path": "/Game/Materials/M_Stone.M_Stone"
}
```

### Errors

| Code | Trigger |
|---|---|
| `missing_required_field` | Any of `parent_path`, `path`, `name` missing/empty. |
| `invalid_path` | `path` doesn't start with `/Game/`. |
| `invalid_asset_name` | `name` contains `/` or `.`. |
| `parent_not_found` | `parent_path` doesn't exist. |
| `parent_not_a_material` | `parent_path` exists but is not a `UMaterialInterface`. |
| `dest_exists` | An asset already exists at the target. |
| `create_failed` | UE asset creation returned null. |

### Implementation

1. Validate params (mirror `create_sequence` from v0.8.0).
2. Load parent: `UMaterialInterface* Parent = Cast<UMaterialInterface>(UEditorAssetLibrary::LoadAsset(parent_path))`. Distinguish "missing" from "wrong type".
3. Build dest path; check `DoesAssetExist` → `dest_exists`.
4. `IAssetTools::CreateAsset(name, path, UMaterialInstanceConstant::StaticClass(), nullptr)` (UE auto-resolves `UMaterialInstanceConstantFactoryNew` by class). Cast result to `UMaterialInstanceConstant*`. If null → `create_failed`.
5. `UMaterialEditingLibrary::SetMaterialInstanceParent(MIC, Parent)` — wires the parent. (Verified at `MaterialEditingLibrary.h:288`.)
6. `UEditorAssetLibrary::SaveAsset(DestObjectPath, false)`.
7. Return success with paths.

## Handler: `set_mi_parameter`

Single handler with a `type` discriminator. The JSON value shape varies by type — the alternative (three separate handlers) inflates the catalog without adding capability.

### Parameters

| Param | Type | Required | Default |
|---|---|---|---|
| `path` | string | Yes | — |
| `parameter` | string | Yes | — |
| `type` | string | Yes | — |
| `value` | varies | Yes | — |

`path` is the `UMaterialInstanceConstant` asset path. `parameter` is the parameter name (case-insensitive in UE). `type` must be one of `"scalar"`, `"vector"`, or `"texture"`. `value` shape depends on `type`:

| `type` | `value` JSON shape |
|---|---|
| `scalar` | number (e.g. `0.75`) |
| `vector` | object `{r, g, b, a}` (each in `[0, 1]`; alpha defaults to `1.0` if omitted) |
| `texture` | string (asset path of a `UTexture`, e.g. `"/Game/Textures/T_Stone_D"`) |

### Result

```json
{
  "ok": true,
  "path": "/Game/Materials/MI_Stone.MI_Stone",
  "parameter": "BaseColorTint",
  "type": "vector",
  "applied_value": {"r": 1.0, "g": 0.5, "b": 0.2, "a": 1.0}
}
```

### Errors

| Code | Trigger |
|---|---|
| `missing_required_field` | Any required param missing. |
| `asset_not_found` | `path` doesn't resolve. |
| `not_a_material_instance` | Asset exists but isn't a `UMaterialInstanceConstant`. |
| `invalid_parameter_type` | `type` is not one of `"scalar"`, `"vector"`, or `"texture"`. |
| `invalid_value_shape` | `value` shape doesn't match `type` (e.g. number passed for a vector). |
| `texture_not_found` | (texture only) `value` doesn't resolve to a `UTexture`. |
| `parameter_not_applied` | UE's `SetMaterialInstance*ParameterValue` returned false (typically: parameter name doesn't exist on the parent material). |

### Implementation

1. Validate params; cast asset to `UMaterialInstanceConstant*`.
2. Switch on `type`:
   - `scalar`: read `value` as number; call `UMaterialEditingLibrary::SetMaterialInstanceScalarParameterValue(MIC, FName(*parameter), Value)` (line 301).
   - `vector`: read `value` as `{r, g, b, a}`; build `FLinearColor`; call `SetMaterialInstanceVectorParameterValue` (line 337).
   - `texture`: read `value` as asset path; load via `UEditorAssetLibrary::LoadAsset` and cast to `UTexture*`; call `SetMaterialInstanceTextureParameterValue` (line 310).
3. Each setter returns bool — false means parameter name didn't match. Emit `parameter_not_applied`.
4. `SaveAsset`.
5. Return.

## Handler: `inspect_material`

Lists the parameter **names** declared by any `UMaterialInterface` (a `UMaterial` base material or a `UMaterialInstance`). Works as the discovery tool: call `find_assets` to find materials, then `inspect_material` to learn what parameters they expose, then `set_mi_parameter` to override.

### Parameters

| Param | Type | Required | Default |
|---|---|---|---|
| `path` | string | Yes | — |

### Result

```json
{
  "ok": true,
  "name": "M_Stone",
  "package_path": "/Game/Materials/M_Stone",
  "class": "Material",
  "scalar_parameters": ["Roughness", "Metallic", "TilingScale"],
  "vector_parameters": ["BaseColorTint"],
  "texture_parameters": ["BaseColorMap", "NormalMap", "RoughnessMap"],
  "static_switch_parameters": ["UseDetailNormal"]
}
```

### Errors

| Code | Trigger |
|---|---|
| `missing_required_field` | `path` missing. |
| `asset_not_found` | Path doesn't resolve. |
| `not_a_material` | Asset exists but isn't a `UMaterialInterface`. |

### Implementation

1. Validate; load via `UEditorAssetLibrary::LoadAsset`; cast to `UMaterialInterface*`. Distinguish "missing" from "wrong type".
2. Call all four `UMaterialEditingLibrary::Get<Type>ParameterNames(Material, OutNames)` (lines 357 / 361 / 365 / 369). Each returns `TArray<FName>`.
3. Convert each array to JSON string array, sort for stable output.
4. Return.

## Handler: `inspect_material_instance`

Reads the **overridden** parameter values from a `UMaterialInstanceConstant`. Complements `inspect_material` (which lists *declared* params on the parent).

### Parameters

| Param | Type | Required | Default |
|---|---|---|---|
| `path` | string | Yes | — |

### Result

```json
{
  "ok": true,
  "name": "MI_Stone",
  "package_path": "/Game/Materials/MI_Stone",
  "parent_path": "/Game/Materials/M_Stone.M_Stone",
  "scalar_overrides": {"Roughness": 0.7, "TilingScale": 4.0},
  "vector_overrides": {"BaseColorTint": {"r": 0.85, "g": 0.85, "b": 0.85, "a": 1.0}},
  "texture_overrides": {"BaseColorMap": "/Game/Textures/T_Stone_D.T_Stone_D"}
}
```

Only **overridden** parameters appear in the output maps. Parameters inherited unchanged from the parent are not listed.

### Errors

| Code | Trigger |
|---|---|
| `missing_required_field` | `path` missing. |
| `asset_not_found` | Path doesn't resolve. |
| `not_a_material_instance` | Asset exists but isn't a `UMaterialInstanceConstant`. |

### Implementation

1. Validate; load; cast to `UMaterialInstanceConstant*`.
2. Read `Instance->Parent` (TObjectPtr; line 647). Stringify via `Parent->GetPathName()`.
3. Iterate `Instance->ScalarParameterValues` (line 750) — each entry has `ParameterInfo.Name` (an `FMaterialParameterInfo`) and `ParameterValue` (float). Build the `scalar_overrides` map.
4. Same for `VectorParameterValues` (line 754) — `ParameterValue` is `FLinearColor`; emit `{r, g, b, a}`.
5. Same for `TextureParameterValues` (line 762) — `ParameterValue` is `UTexture*`; emit asset path or empty string for null.

## Build.cs change

Add `MaterialEditor` to `PrivateDependencyModuleNames`:

```csharp
PrivateDependencyModuleNames.AddRange(new string[] {
    // ... existing ...
    // Material instance authoring (v0.9.0)
    "MaterialEditor"
});
```

`MaterialEditor` is the module that ships `UMaterialEditingLibrary`. It's editor-only (matches our plugin's `Type: Editor` declaration) and pulls no shipping-build risk.

Other UE module deps the handlers need (`Engine` for `UMaterialInstanceConstant`, `AssetTools` for `IAssetTools::CreateAsset`, `EditorScriptingUtilities` for `UEditorAssetLibrary`, `UnrealEd` for the factory class) are already in Build.cs from earlier bundles.

Purely additive. All v0.4.0–v0.8.0 handlers continue to work unchanged. The four new handlers are new MCP methods. The Build.cs addition is a single private module — no impact on existing compilation.

## Risk surface and mitigations

| Risk | Mitigation |
|---|---|
| `SetMaterialInstance*ParameterValue` returns false silently when the parameter name doesn't exist on the parent | Handler captures the bool return and emits `parameter_not_applied` rather than reporting success. (Codex caught the same pattern in v0.6.0's `execute_console_command` — applying the lesson here proactively.) |
| `inspect_material` parameter-name lists may include layer/blend parameters that don't apply to GlobalParameter overrides | UE's `Get<Type>ParameterNames` accepts a `UMaterialInterface*` and returns global parameters by default. Document that v0.9.0 doesn't surface layer parameters. |
| `UMaterialInstance::Parent` may be a `UMaterialInstance` itself, not a `UMaterial` | That's fine — UE supports MI-on-MI parenting. Spec's `parent_path` and the inspect output both use `UMaterialInterface` semantics. |
| Setting a vector value via `FLinearColor` clamps to `[0, 1]` for some parameter types | UE's `FLinearColor` is unbounded; the clamp happens in the shader. We pass values through unchanged and let the shader handle range. |
| Texture parameter assignment may require the texture to be saved | We use `UEditorAssetLibrary::LoadAsset` which loads from disk. If the asset is unsaved/in-memory the path won't resolve and we emit `texture_not_found`. |
| Parameter name comparison is case-sensitive in UE 5.7's FName lookup but case-insensitive at the editor UI level | Document: parameter names are `FName`-compared. Pass them exactly as they appear in `inspect_material`'s output. |
