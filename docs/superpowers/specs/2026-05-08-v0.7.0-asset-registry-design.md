# UnrealClaudeMCP v0.7.0 — Asset registry deep dive

**Date:** 2026-05-08
**Status:** Approved (autonomous-mode design — user-authorized "do everything" execution; spec stands as the design contract)
**Author:** Claude (with Najem)
**Target release:** v0.7.0
**Builds on:** v0.6.0 (21 handlers; observability)
**Scope:** Tag-aware querying + full asset-lifecycle authoring. One handler extended (`find_assets` gains tag filtering + tag inclusion in output) and four new handlers (`inspect_asset`, `move_asset`, `rename_asset`, `delete_asset`).

---

## Goal

Give the LLM **complete editor-side asset-management authority**: query by metadata tags, read every fact the asset registry knows about an asset (class, tags, dependencies, referencers, package size, source files), and mutate paths/names/existence with appropriate safety checks.

Tool count: 21 → 25. Version: 0.6.0 → 0.7.0.

These tools are write-capable. `delete_asset` is the highest-impact handler in the plugin to date — losing a referenced texture can cascade across an entire project. The design therefore puts a referencer-check safety gate in front of every destructive call (force-flag override available). `move_asset` and `rename_asset` cannot lose data (UE auto-creates redirectors), so they don't need the same gate.

---

## Scope

### In scope

- `find_assets` extension: `tags` parameter + `include_tags` output toggle.
- `inspect_asset` handler — read-only: class, tags, dependencies, referencers, package metadata.
- `move_asset` handler — change an asset's containing folder; leaf name unchanged.
- `rename_asset` handler — change an asset's leaf name; folder unchanged.
- `delete_asset` handler — delete an asset; refuses if referenced unless `force=true`.
- All version bumps (.uplugin, mcp_manifest.json, bridge, README).
- Bridge catalog + JSON Schema for the four new tools and the schema diff on `find_assets`.
- Bridge unit tests: schema-shape tests for the four new tools and `find_assets` schema additions, plus manifest-sync count assertions.
- Smoke test additions: 4 new steps + 1 modified step (`find_assets` with tags).
- `docs/TOOLS.md` additions for all four new tools and the `find_assets` extension.
- Spec doc (this file).

### Out of scope

- Bulk operations (`bulk_delete_assets`, `bulk_move_assets`, etc.) — deferred to v0.7.x if single-call perf becomes a measured pain point. Bulk error handling (partial-success, transactional rollback) is non-trivial and is not justified by current LLM workflows.
- `duplicate_asset` — useful but lower priority than the four chosen handlers; can be added in v0.7.x without breaking v0.7.0.
- Folder-level operations (`delete_directory`, `rename_directory`) — `EditorAssetLibrary` exposes these, but they're amplifiers of the same risk surface as `delete_asset`/`rename_asset` and add tooling-permission complexity. Defer.
- Source-control integration (`bAutoCheckout` flag) — UE's `RenameAsset` already attempts checkout on its own when SCC is enabled; we don't need to expose the flag.
- New tag *writes* on assets — runtime tag mutation requires editing the asset's source object (e.g. setting an FName meta-tag inside a Blueprint), which is per-asset-class and out of scope here.

## Handler: `find_assets` (extension)

The existing `find_assets` (v0.4.0) is augmented with two additive params and one optional output field. The original 4-param shape (`class_path`, `path_under`, `name_contains`, `limit`) continues to work unchanged.

### New parameters

| Param | Type | Required | Default | Purpose |
|---|---|---|---|---|
| `tags` | object | No | — | Map of tag-name → required-value (string) or `null` (any value). Multiple entries are AND-combined. |
| `include_tags` | bool | No | `false` | When true, each result asset includes a `tags` map of all its registry tags. |

### Result (when `include_tags=true`)

```json
{
  "ok": true,
  "matched": 12,
  "returned": 12,
  "assets": [
    {
      "name": "T_Stone_D",
      "package_path": "/Game/Textures/Environment/T_Stone_D",
      "class": "Texture2D",
      "tags": {
        "CompressionSettings": "TC_Default",
        "LODGroup": "TEXTUREGROUP_World",
        "SourceFile": "..."
      }
    }
  ]
}
```

### New errors

| Code | Trigger |
|---|---|
| `invalid_tag_value` | A `tags` map entry's value is not a string or null. |

### Implementation

`tags` is appended to `FARFilter::TagsAndValues` (a `TMultiMap<FName, TOptional<FString>>`). Empty `TOptional` = "any value" semantics. Multiple AND-combined entries are added as separate map entries; UE's asset registry treats them as ANDed automatically.

When `include_tags=true`, the per-asset loop reads `FAssetData::TagsAndValues` (a `FAssetTagValueRef` map) and serializes each entry as `(string tag-name) -> (string tag-value)`. Numeric or object-path tag values are stringified via `FAssetData::GetTagValueRef<FString>()`.

## Handler: `inspect_asset`

### Parameters

| Param | Type | Required | Default |
|---|---|---|---|
| `path` | string | Yes | — |

`path` accepts either form: `/Game/Folder/Name` (package path) or `/Game/Folder/Name.Name` (asset path with object suffix). The handler normalizes internally.

### Result

```json
{
  "ok": true,
  "name": "T_Stone_D",
  "package_path": "/Game/Textures/Environment/T_Stone_D",
  "asset_path": "/Game/Textures/Environment/T_Stone_D.T_Stone_D",
  "class": "Texture2D",
  "class_path": "/Script/Engine.Texture2D",
  "tags": {
    "CompressionSettings": "TC_Default",
    "LODGroup": "TEXTUREGROUP_World"
  },
  "dependencies": [
    "/Game/Materials/M_Stone"
  ],
  "referencers": [
    "/Game/Maps/MyMap"
  ],
  "package_size_bytes": 487293
}
```

### Errors

| Code | Trigger |
|---|---|
| `missing_required_field` | `path` missing or empty. |
| `asset_not_found` | No asset at the given path. |

### Implementation

1. Normalize `path` (strip `.Name` suffix if present to get the package path).
2. `FAssetRegistryModule::Get().GetAssetByObjectPath(FSoftObjectPath(path))` (UE 5.7 returns a single `FAssetData`).
3. Read `AssetName`, `PackageName`, `AssetClassPath`, `TagsAndValues`.
4. `Registry.GetDependencies(PackageName, OutDeps)` and `Registry.GetReferencers(PackageName, OutRefs)`.
5. Read package size from disk: `FFileManagerGeneric::Get().FileSize(FPackageName::LongPackageNameToFilename(PackageName, FPackageName::GetAssetPackageExtension()))`. If file is unsaved/in-memory, returns -1; we'll surface `null` in JSON.
6. Build the JSON result.

## Handler: `move_asset`

### Parameters

| Param | Type | Required | Default |
|---|---|---|---|
| `path` | string | Yes | — |
| `dest_folder` | string | Yes | — |

`path` is the source asset path (object-path or package-path form). `dest_folder` is a destination folder under `/Game/` or `/Engine/`, e.g. `/Game/Textures/Stone/`.

### Result

```json
{
  "ok": true,
  "old_path": "/Game/Textures/T_Stone_D.T_Stone_D",
  "new_path": "/Game/Textures/Environment/T_Stone_D.T_Stone_D"
}
```

### Errors

| Code | Trigger |
|---|---|
| `missing_required_field` | `path` or `dest_folder` missing/empty. |
| `asset_not_found` | Source asset does not exist. |
| `invalid_dest_folder` | `dest_folder` does not start with `/Game/` or `/Engine/`. |
| `dest_exists` | An asset of the same name already exists at the destination. |
| `rename_failed` | UE's `RenameAsset` returned false (SCC checkout failure, etc). |

### Implementation

1. Normalize source `path` to full asset path.
2. Validate `dest_folder` starts with `/Game/` or `/Engine/`; trim trailing slash.
3. Build dest path as `dest_folder + AssetName + "." + AssetName`.
4. If `EditorAssetLibrary::DoesAssetExist(dest_path)` → `dest_exists` error.
5. Call `EditorAssetLibrary::RenameAsset(source_path, dest_path)`.
6. UE auto-creates a redirector at the source path so existing references continue to resolve.

## Handler: `rename_asset`

### Parameters

| Param | Type | Required | Default |
|---|---|---|---|
| `path` | string | Yes | — |
| `new_name` | string | Yes | — |

`new_name` is a leaf name only — no `/` and no `.` allowed.

### Result

```json
{
  "ok": true,
  "old_path": "/Game/Textures/T_Stone.T_Stone",
  "new_path": "/Game/Textures/T_StoneAlbedo.T_StoneAlbedo"
}
```

### Errors

| Code | Trigger |
|---|---|
| `missing_required_field` | `path` or `new_name` missing/empty. |
| `asset_not_found` | Source asset does not exist. |
| `invalid_asset_name` | `new_name` contains `/` or `.`, or is empty after trim. |
| `dest_exists` | An asset with `new_name` already exists in the same folder. |
| `rename_failed` | UE's `RenameAsset` returned false. |

### Implementation

1. Normalize source `path`.
2. Validate `new_name` (no path separators, non-empty).
3. Compute source folder = `path` minus `Name.Name` suffix.
4. Build dest path as `source_folder + "/" + new_name + "." + new_name`.
5. If `DoesAssetExist(dest)` → `dest_exists` error.
6. Call `EditorAssetLibrary::RenameAsset(source_path, dest_path)`.

`move_asset` and `rename_asset` share an internal helper `RenameAssetByFullPaths(Source, Dest)` that handles the `DoesAssetExist` check and the `RenameAsset` call. Each public handler is a thin parameter-shape wrapper. This lets the LLM pick the right tool for each mental model — moving (folder change) and renaming (leaf change) are conceptually distinct even though they share an underlying op.

## Handler: `delete_asset`

### Parameters

| Param | Type | Required | Default |
|---|---|---|---|
| `path` | string | Yes | — |
| `force` | bool | No | `false` |

When `force=false`, the handler refuses to delete an asset that has any external referencers and reports them via `has_referencers`. When `force=true`, the deletion proceeds regardless.

### Result

```json
{
  "ok": true,
  "deleted_path": "/Game/Textures/T_Stone_OldVariant.T_Stone_OldVariant"
}
```

### Errors

| Code | Trigger |
|---|---|
| `missing_required_field` | `path` missing or empty. |
| `asset_not_found` | Source asset does not exist. |
| `has_referencers` | Asset is referenced by N other packages and `force=false`. The error message lists up to 5 referencer package names. |
| `delete_failed` | UE's `DeleteAsset` returned false (file lock, SCC failure, etc). |

### Implementation

1. Normalize `path` to package name.
2. `FAssetRegistry::GetReferencers(PackageName, OutRefs)`. UE returns dependency-graph referencers — packages that hold a hard reference to this one.
3. If `OutRefs.Num() > 0` and `force=false`: emit `has_referencers` with the list.
4. Call `EditorAssetLibrary::DeleteAsset(path)`. UE's documentation explicitly notes this is a *force-delete* with no built-in safety check ("It doesn't check if the asset has references in other Levels or by Actors") — that's why we ran the safety check in step 2 above.
5. Return success.

## New stable error codes

| Code | Handler | Trigger |
|---|---|---|
| `invalid_tag_value` | `find_assets` | A `tags` map entry's value is not a string or null. |
| `asset_not_found` | `inspect_asset`, `move_asset`, `rename_asset`, `delete_asset` | Source path does not resolve to an asset. |
| `invalid_dest_folder` | `move_asset` | `dest_folder` does not start with `/Game/` or `/Engine/`. |
| `dest_exists` | `move_asset`, `rename_asset` | Destination already has an asset with this name. |
| `rename_failed` | `move_asset`, `rename_asset` | UE's `RenameAsset` returned false. |
| `invalid_asset_name` | `rename_asset` | `new_name` contains `/` or `.`, or is empty. |
| `has_referencers` | `delete_asset` | Asset is referenced and `force=false`. |
| `delete_failed` | `delete_asset` | UE's `DeleteAsset` returned false. |

These join the stable set established in v0.2.0–v0.6.0.

## Backward compatibility

Purely additive at the protocol level. Every v0.4.0/v0.6.0 tool call continues to work unchanged:

- `find_assets` is **non-breaking**: `tags` and `include_tags` are optional; without them the handler behaves identically to v0.4.0. Result schema is **additively extended** — `tags` per-asset is only present when `include_tags=true`.
- The four new handlers are new MCP methods; existing tools are unaffected.

There is no on-disk format change. The asset registry is read-through; mutations go through UE's canonical asset-tools APIs that already handle the .uasset format.

## Risk surface and mitigations

| Risk | Mitigation |
|---|---|
| `DeleteAsset` is force-delete by default (no UE-side referencer check) | We do the referencer check ourselves via `IAssetRegistry::GetReferencers` before calling DeleteAsset. `force=true` opts out. |
| `RenameAsset` may show modal dialogs in some SCC configurations | Document this; advise users running source-control to test in a scratch folder first. UE's `bAutoCheckout` parameter is a different API path (`IAssetTools::RenameAssetsWithDialog`) that we explicitly avoid. |
| `RenameAsset` creates redirectors that never get cleaned up | This is UE's standard behavior; document it. Future v0.7.x could add a `fix_up_redirectors` handler that wraps `IAssetTools::FixupReferencers`. |
| `inspect_asset` on a transient/in-memory asset returns -1 file size | Return JSON `null` for `package_size_bytes` when the file is not on disk. |
| Tag filter case sensitivity | UE asset-registry tag *names* are FNames (case-insensitive comparison). Tag *values* are FStrings (case-sensitive). Document this. |
| Asset-registry initialization race on editor startup | The asset registry must be fully scanned before queries return reliable results. UE 5.7 fires `IAssetRegistry::OnFilesLoaded()` when the initial scan completes. Our handlers run on the game thread inside the editor; by the time the MCP server is dispatching, the registry has been scanning for several seconds and is usually complete. If a query hits during partial scan we'll silently under-return — acceptable for v0.7.0; document. |
