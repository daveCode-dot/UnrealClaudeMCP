# UnrealClaudeMCP v0.8.0 — Sequencer scaffolding

**Date:** 2026-05-08
**Status:** Approved (autonomous-mode design — user-authorized "do everything" execution; spec stands as the design contract)
**Author:** Claude (with Najem)
**Target release:** v0.8.0
**Builds on:** v0.7.0 (25 handlers; asset registry deep dive)
**Scope:** Three new MCP handlers for Level Sequence introspection and scaffolding. Keyframe authoring (templated channel APIs) and Movie Render Queue (separate subsystem) are explicitly out of scope and remain accessible via `execute_unreal_python`.

---

## Goal

Give the LLM **read access to existing cinematics + the ability to scaffold new ones**: discover sequences via `find_assets`, read their structure (tracks, sections, bindings, time range) via `inspect_sequence`, create new empty sequences via `create_sequence`, and bind level actors to them as possessables via `bind_actor_to_sequence`.

Tool count: 25 → 28. Version: 0.7.0 → 0.8.0.

The deliberate omission of keyframe authoring is the cost of staying within reasonable bundle size. UE's `IMovieSceneChannelDataProvider` / `FMovieSceneFloatChannel::AddCubicKey` / `FMovieSceneFloatChannel::AddLinearKey` family is heavily templated, type-erased through interface tables, and has subtle interpolation-mode pitfalls. Wrapping it in a single MCP-friendly handler is a v0.8.x project on its own. For now, the LLM authors keyframes via `execute_unreal_python` (which has full access to `unreal.MovieSceneFloatChannel.add_key`, etc.), then re-inspects via `inspect_sequence` to verify.

---

## Scope

### In scope

- `inspect_sequence` handler — read tracks, sections, bindings, time range from a `ULevelSequence`.
- `create_sequence` handler — create a new `ULevelSequence` asset at a `/Game/...` path.
- `bind_actor_to_sequence` handler — add a level actor as a possessable to an existing sequence.
- Build.cs additions: `LevelSequence`, `MovieScene`, `MovieSceneTracks` (public) + `LevelSequenceEditor` (private). First Build.cs change since v0.4.0.
- Version bumps (.uplugin, mcp_manifest.json, bridge, README).
- Bridge catalog + JSON Schema for the three new handlers.
- Bridge unit tests: schema-shape tests + manifest sync count.
- Smoke step: read-only inspection of a sequence asset (or warn-and-skip if no sequences in project).
- `docs/TOOLS.md` additions.
- Spec doc (this file).

### Out of scope

- Keyframe authoring (`add_transform_keyframe`, etc.) — channel APIs are templated; deferred to v0.8.x. Use `execute_unreal_python` for now.
- Movie Render Queue / `render_sequence_to_disk` — distinct subsystem (`MovieRenderPipelineCore`, `MovieRenderPipelineSettings`); deferred to v0.8.x or a separate v0.8.0/v0.8.1 split if scope budget allows.
- Spawnable bindings — `ULevelSequence::AddSpawnable` requires a template object and creates a per-sequence-instance actor. v0.8.0 only supports possessables (existing actors); spawnables are v0.8.x.
- Sub-sequences and shot tracks — niche workflow, deferred.
- Camera Cuts and master tracks — could be added in v0.8.x via a `add_track_to_sequence(type)` handler; v0.8.0 stays focused on the core scaffolding.
- Removal/deletion of bindings or tracks — deferred.

## Handler: `inspect_sequence`

### Parameters

| Param | Type | Required | Default |
|---|---|---|---|
| `path` | string | Yes | — |

`path` is a Level Sequence asset path (object path or package path).

### Result

```json
{
  "ok": true,
  "name": "MainCinematic",
  "package_path": "/Game/Cinematics/MainCinematic",
  "tick_resolution": 24000,
  "display_rate_fps": 24.0,
  "playback_range": { "start_frames": 0, "end_frames": 240 },
  "bindings": [
    {
      "guid": "1A2B3C4D-...",
      "name": "MyHero",
      "type": "possessable",
      "bound_actor_label": "MyHero_Actor"
    }
  ],
  "tracks": [
    {
      "name": "Transform",
      "class": "MovieScene3DTransformTrack",
      "section_count": 1,
      "binding_guid": "1A2B3C4D-..."
    }
  ]
}
```

### Errors

| Code | Trigger |
|---|---|
| `missing_required_field` | `path` missing/empty. |
| `asset_not_found` | Path doesn't resolve to any asset. |
| `not_a_sequence` | Asset exists but is not a `ULevelSequence`. |

### Implementation

1. Validate `path`.
2. Use `UEditorAssetLibrary::LoadAsset(ObjectPath)` (or `LoadObject<ULevelSequence>`) to get the `ULevelSequence*`.
3. Cast to `ULevelSequence*`. If null: `not_a_sequence`.
4. Call `Sequence->GetMovieScene()` to get the `UMovieScene*`. (Confirmed at `LevelSequence.h:51` of UE 5.7.)
5. Read tick resolution (`MovieScene->GetTickResolution()`), display rate (`MovieScene->GetDisplayRate()`), playback range (`MovieScene->GetPlaybackRange()`).
6. Loop `MovieScene->GetSpawnableCount()` and `MovieScene->GetPossessableCount()`, build `bindings` array. Each binding has GUID + name; for possessables, look up the bound object via `Sequence->LocateBoundObjects(Guid, ResolutionContext)` to get the actor label (best-effort — may be null if the binding can't be resolved without a world context).
7. Loop master tracks via `MovieScene->GetMasterTracks()` and binding-attached tracks via `MovieScene->GetTracks(Binding)`. Each track has class name + section count.

## Handler: `create_sequence`

### Parameters

| Param | Type | Required | Default |
|---|---|---|---|
| `path` | string | Yes | — |
| `name` | string | Yes | — |
| `display_rate_fps` | number | No | `30.0` |
| `playback_end_frames` | int | No | `240` (~8s @ 30fps) |

`path` is the destination folder under `/Game/`. `name` is the asset's leaf name.

### Result

```json
{
  "ok": true,
  "asset_path": "/Game/Cinematics/MyNewSequence.MyNewSequence",
  "package_path": "/Game/Cinematics/MyNewSequence",
  "display_rate_fps": 30.0,
  "playback_range": { "start_frames": 0, "end_frames": 240 }
}
```

### Errors

| Code | Trigger |
|---|---|
| `missing_required_field` | `path` or `name` missing/empty. |
| `invalid_path` | `path` doesn't start with `/Game/`. |
| `invalid_asset_name` | `name` contains `/` or `.`. |
| `dest_exists` | An asset already exists at the target. |
| `create_failed` | Asset creation returned null. |

### Implementation

1. Validate params (name passes `UCMCPAssetPath::IsValidLeafName`).
2. Build target object path: `path + "/" + name + "." + name`.
3. Check `UEditorAssetLibrary::DoesAssetExist` — if true, `dest_exists`.
4. Use `UAssetToolsModule::Get().Get().CreateAsset(name, path, ULevelSequence::StaticClass(), nullptr)` to create the asset. (`UAssetTools::CreateAsset` is the canonical creation path.) The factory is auto-resolved if `nullptr` is passed; UE matches the factory by class.
5. Cast to `ULevelSequence*`. If null: `create_failed`.
6. Configure: `Sequence->Initialize()` (creates the underlying `UMovieScene`); `MovieScene->SetDisplayRate(FFrameRate(display_rate_fps, 1))`; `MovieScene->SetPlaybackRange(0, playback_end_frames)`.
7. Save the asset: `UEditorAssetLibrary::SaveAsset(ObjectPath, false)`.
8. Return success with the asset path + final config.

## Handler: `bind_actor_to_sequence`

### Parameters

| Param | Type | Required | Default |
|---|---|---|---|
| `sequence_path` | string | Yes | — |
| `actor_name` | string | Yes | — |

`sequence_path` is the Level Sequence asset path. `actor_name` is the actor's label or FName in the current editor world (same hybrid identification as `set_actor_property` etc. from v0.3.0/v0.4.0).

### Result

```json
{
  "ok": true,
  "sequence_path": "/Game/Cinematics/MainCinematic.MainCinematic",
  "binding_guid": "1A2B3C4D-...",
  "actor_label": "MyHero_Actor",
  "binding_type": "possessable"
}
```

### Errors

| Code | Trigger |
|---|---|
| `missing_required_field` | `sequence_path` or `actor_name` missing/empty. |
| `asset_not_found` | Sequence path doesn't resolve. |
| `not_a_sequence` | Asset exists but isn't a `ULevelSequence`. |
| `actor_not_found` | No actor in the editor world matches the label/FName. |
| `ambiguous_actor` | Multiple actors share the label and no FName disambiguator was provided. (Mirrors v0.3.0 hybrid identification.) |
| `bind_failed` | `AddPossessable` returned an invalid GUID. |

### Implementation

1. Validate params.
2. Load sequence (same as `inspect_sequence` step 2-3).
3. Resolve actor by label or FName in the editor world (helper used by other v0.3.0+ handlers).
4. Call `Sequence->GetMovieScene()->AddPossessable(ActorLabel, Actor->GetClass())`. Returns an FGuid; if invalid, `bind_failed`.
5. Call `Sequence->BindPossessableObject(Guid, *Actor, World)` to wire the GUID to the live actor.
6. Save the sequence asset.
7. Return GUID + binding type.

## Build.cs changes

First Build.cs change since v0.4.0. Adds three public + one private dependency:

```csharp
PublicDependencyModuleNames.AddRange(new string[] {
    // ... existing ...
    "LevelSequence",        // ULevelSequence, runtime
    "MovieScene",           // UMovieScene, FMovieSceneBinding, FFrameRate
    "MovieSceneTracks",     // built-in track classes (UMovieScene3DTransformTrack, etc.)
});

PrivateDependencyModuleNames.AddRange(new string[] {
    // ... existing ...
    "LevelSequenceEditor",  // ULevelSequenceEditorBlueprintLibrary (for open-in-editor — optional, used only if we need it)
});
```

`LevelSequenceEditor` is private because we don't expose its types in handler-public headers; we just call into it from `.cpp`.

## Backward compatibility

Purely additive. All v0.4.0–v0.7.0 handlers continue to work unchanged. The three new handlers are new MCP methods. The Build.cs additions don't affect compilation of existing code (additive module dependencies).

## Risk surface and mitigations

| Risk | Mitigation |
|---|---|
| `LevelSequenceEditor` private dependency may pull in editor-only headers that break shipping builds | Plugin is already `Type: Editor` in the .uplugin, so this is editor-only by definition. No shipping build risk. |
| `LocateBoundObjects` may return null in headless contexts | We guard for null and return the GUID-only binding without `bound_actor_label` if resolution fails. |
| `UAssetTools::CreateAsset` triggers modal dialogs in some configurations | Use the non-dialog overload that takes a factory directly. v0.3.0's `Handler_ImportTexture` has the same modal-suppression pattern; mirror it. |
| `AddPossessable` requires the binding be subsequently wired via `BindPossessableObject` to actually resolve | Both calls are in the handler; they're not separate user-facing operations. |
| Sequencer asset registry tags may not be populated immediately after creation | After `CreateAsset`, call `Sequence->MarkPackageDirty()` and `UEditorAssetLibrary::SaveAsset()` so the asset registry sees it. |
| Frame numbers vs. time-in-seconds mismatch | All public params/results use frame numbers in display rate. Internal conversion uses `FFrameRate::AsFrameNumber`. Documented. |
