# UnrealClaudeMCP v0.6.0 — Observability design

**Date:** 2026-05-08
**Status:** Approved — proceeding to implementation
**Author:** Claude (with Najem)
**Target release:** v0.6.0
**Builds on:** v0.4.0 (19 handlers; v0.5.0 is a parallel PR touching wire-protocol only)
**Scope:** Two new MCP handlers giving the LLM read access to the UE Output Log and the ability to execute console commands.

---

## Goal

Give the LLM **eyes + voice in the editor** — two complementary capabilities that enable closed-loop debugging:

1. **`get_log_lines`** — read recent UE Output Log entries with category and verbosity filters.
2. **`execute_console_command`** — run a UE console command and capture its output.

Tool count: 19 → 21. Version: 0.4.0 → 0.6.0 (0.5.0 is a parallel wire-framing PR; this branch diverged before 0.5.0 merged, so we skip that number on this branch).

These two tools are deliberately **read-heavy / safe**: `get_log_lines` is purely read-only; `execute_console_command` runs console commands (which can mutate editor state) but exposes no C++ path to file-system writes or asset deletion beyond what UE's own console already allows.

## Handler: `get_log_lines`

### Parameters

| Param | Type | Required | Default |
|---|---|---|---|
| `count` | int | No | 100 (cap 1000) |
| `category_filter` | string | No | — (all categories) |
| `min_verbosity` | string | No | `"Log"` |

`min_verbosity` accepts: `"Fatal"`, `"Error"`, `"Warning"`, `"Display"`, `"Log"`, `"Verbose"`, `"VeryVerbose"`. Lines at or above the specified severity are returned. The UE severity ladder (highest to lowest) is: `Fatal(0) > Error(1) > Warning(2) > Display(3) > Log(4) > Verbose(5) > VeryVerbose(6)`. "At or above" means numerically ≤ the requested level.

### Result

```json
{
  "ok": true,
  "returned": 42,
  "lines": [
    {
      "timestamp": "2026.05.08-13.30.45",
      "category": "LogTemp",
      "verbosity": "Warning",
      "message": "something unexpected happened"
    }
  ]
}
```

### Errors

| Code | Trigger |
|---|---|
| `invalid_verbosity` | `min_verbosity` string is not one of the seven accepted values. |

### Implementation

Reads from `FUCMCPLogCapture::Get()` (see shared module below). The ring buffer is copied to a working array under the mutex; filtering is done outside the lock. `category_filter` is a case-insensitive substring match on the category name. `count` cap is applied after filtering (returns the N most-recent matching lines).

## Handler: `execute_console_command`

### Parameters

| Param | Type | Required | Default |
|---|---|---|---|
| `command` | string | Yes | — |
| `capture_output` | bool | No | `true` |

### Result

```json
{
  "ok": true,
  "command": "stat fps",
  "output": "Frame: 16.6ms (60 FPS)..."
}
```

When `capture_output=false`, `output` is an empty string. The command still executes; the result just doesn't capture its console text.

### Errors

| Code | Trigger |
|---|---|
| `missing_required_field` | `command` param missing or empty. |
| `command_execution_failed` | `GEngine` or `GEditor` is null, or `Exec` returned false. |

### Implementation

1. Verify `GEditor` is non-null; get world via `GEditor->GetEditorWorldContext().World()`.
2. If `capture_output=true`: create a `FStringOutputDevice` on the stack. Call `GEngine->Exec(World, *Command, OutputDevice)`.
3. Read the captured output from `FStringOutputDevice` and return it.
4. If `capture_output=false`: pass `*GLog` (or a null output device) as the output device so console output flows to the normal Output Log.

`FStringOutputDevice` is declared in `Engine/Source/Runtime/Core/Public/Misc/OutputDeviceString.h` (UE 5.7). It is a concrete subclass of `FOutputDevice` that accumulates all `Serialize` calls into a single `FString` member (`OutputString`). This is exactly the right tool for capturing exec output.

## Shared module: `FUCMCPLogCapture`

### Files

- `Source/UnrealClaudeMCP/Private/MCP/LogCapture.h`
- `Source/UnrealClaudeMCP/Private/MCP/LogCapture.cpp`

### Design

```cpp
// Ring buffer size (default, adjustable via constant)
static constexpr int32 kLogCaptureRingSize = 1000;

struct FUCMCPLogEntry
{
    FString Timestamp;   // "YYYY.MM.DD-HH.MM.SS"
    FString Category;    // e.g. "LogTemp"
    FString Verbosity;   // e.g. "Warning"
    FString Message;     // raw message text
};

class FUCMCPLogCapture : public FOutputDevice
{
public:
    static FUCMCPLogCapture& Get();

    // FOutputDevice overrides
    virtual void Serialize(const TCHAR* V, ELogVerbosity::Type Verbosity,
                           const FName& Category) override;

    // Read a snapshot of the ring buffer.
    TArray<FUCMCPLogEntry> GetLines() const;

private:
    FUCMCPLogCapture();

    mutable FCriticalSection Mutex;
    TArray<FUCMCPLogEntry> Ring;   // fixed-capacity ring buffer
    int32 Head = 0;                 // next write position (wraps)
    int32 Count = 0;                // entries filled so far (capped at kLogCaptureRingSize)
};
```

### `Serialize` signature

UE 5.7 `FOutputDevice::Serialize` has two overloads (from `Engine/Source/Runtime/Core/Public/Misc/OutputDevice.h`):

```cpp
// Primary overload — category + verbosity
virtual void Serialize(const TCHAR* V, ELogVerbosity::Type Verbosity, const FName& Category);

// Legacy overload without category (for compatibility with old code)
virtual void Serialize(const TCHAR* V, ELogVerbosity::Type Verbosity, const FName& Category, double Time);
```

The handler overrides the four-parameter version (with `Time`) since that is the one actually called by `FOutputDeviceRedirector` when dispatching to registered output devices. The three-parameter version defaults to calling the four-parameter one with `FPlatformTime::Seconds()`. We override the four-parameter version to avoid double-dispatch.

UE source reference: `Engine/Source/Runtime/Core/Public/Misc/OutputDevice.h` lines ~100-120 (UE 5.7).

### Thread safety

UE's log system (`FOutputDeviceRedirector`) calls registered output devices from **any thread** (render thread, worker threads, game thread). The ring buffer write (`Head`, `Count`, `Ring[Head]`) is protected by `FCriticalSection Mutex`. The `GetLines()` reader takes the same lock to copy the ring to a local array, then releases before returning. Filtering happens outside the lock.

### Registration

In `FUnrealClaudeMCPModule::StartupModule`:
```cpp
GLog->AddOutputDevice(&FUCMCPLogCapture::Get());
```

In `FUnrealClaudeMCPModule::ShutdownModule` (after `FUCMCPServer::Get().Stop()`):
```cpp
GLog->RemoveOutputDevice(&FUCMCPLogCapture::Get());
```

`GLog` is `FOutputDeviceRedirector*` (the global log multiplexer). `AddOutputDevice` / `RemoveOutputDevice` are thread-safe on UE's side.

## Backward compatibility

Purely additive. Every call from v0.4.0 continues to work unchanged. The two new tools are new MCP methods; existing tools are unaffected. The LogCapture singleton begins buffering as soon as `StartupModule` runs; if `get_log_lines` is never called, it has zero runtime effect beyond the mutex overhead on every log line (negligible).
